Two unrelated cleanups landing together.

**1. Reconciler enumeration filter unified across all sites.**

End-user post-clear-cache sync log showed `imputation.totalPhone=7502` but `full-enum.total=6946` — a 556-asset gap. Root cause: three PHAsset.fetchAssets sites in `PhotoKitPersistentChangeReconciler.swift` were still using narrower filters (`.typeUserLibrary` only, hidden excluded) than what `runImputationPass` and `performLiveReconciliation`'s `visibleFetch` adopted at builds 116/121:

- `runIncremental`'s id-existence check (line 1190): `[.typeUserLibrary, .typeCloudShared]` — missing iTunesSynced
- `hashAllCurrentAssets`'s full-enum fetch (line 1614): `[.typeUserLibrary]` only, hidden excluded
- `discoverUntrackedAssets` (line 2567): `[.typeUserLibrary]` only, hidden excluded
- `reconcileCacheAgainstLibrary` orphan sweep (line 2604): `[.typeUserLibrary, .typeCloudShared]` — missing iTunesSynced

All four broadened to `[.typeUserLibrary, .typeCloudShared, .typeiTunesSynced]` + `includeHiddenAssets = true`, matching the imputation + alive-on-phone enumeration. Net effect: hidden + iTunes-synced assets now flow through every reconciliation path consistently — no more "imputed but never hashed" gap, no more spurious "untracked" reports for those source types.

**2. Diagnostic log flusher moved off MainActor — fixes the recurring UI stutter during heavy sync.**

`DiagnosticLogFlusher` was a `@MainActor` class. Its 20-second periodic poll calls `OSLogStore.getEntries(at:matching:)` (which during heavy sync-time logging can take dozens of ms reading the system log buffer), then loops over the collected entries to filter and format them as strings. All of that was running on the main thread. The result was a visible UI stutter every ~20s while a sync was active.

Refactored to a Swift `actor` so the poll work runs on its own executor. No behavior change to capture coverage — same predicate, same cadence, same persistent-store actor downstream. Just keeps main free for the UI to scroll, animate, and respond to taps without the periodic interruption.

Three call sites updated to await the now-actor-isolated methods (`CairnApp.init`, the scene-phase-background flush, and `LogExporter.export`). All existing call sites were already on async paths, so the change is local.
