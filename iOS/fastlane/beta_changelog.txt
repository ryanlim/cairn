Reshaped Settings into a hierarchical layout. The old long single-scroll page is now a flat list of category rows (Connection, Library, Safety & limits, Notifications, Permissions, Appearance, Data, Recovery, Advanced, Danger zone, About) that drill into sub-pages via standard iOS push navigation. Bucketing follows the user's task, not the underlying data model — e.g. Library combines indexing scope + initial scan + excluded assets; Recovery combines find-missed-deletions + deferred queue + rescan + clear hash cache.

The Connection row surfaces a status summary inline ("Connected" / "Offline") and the Library row shows the current scope ("All photos" / "N albums") so a misconfigured setup is visible without drilling in. Phase 1 is mechanical — no copy or behavior changes, just a layout reshape. Phase 2 will add a Quick settings card at the top + more inline summaries on the parent rows; Phase 3 adds search across Settings.
