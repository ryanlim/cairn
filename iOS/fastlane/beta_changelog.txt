In-app diagnostic log export for external testers.

Settings → Advanced → "Export diagnostic logs" now queries the system log store for cairn's recent entries (subsystems `app.cairn` and `app.cairn.ios`), formats them into a plain-text file in cairn's temp directory, and presents the system share sheet. The tester can then email, AirDrop, or save the file to share with support.

File header carries the marketing version, build number, device model (raw identifier — `iPhone15,2` etc), and iOS version so the receiver doesn't have to ask. Body is sorted oldest-first, one line per entry: `HH:mm:ss.fff [<category>] <message>`. Time window defaults to 48 hours.

Important scope limitation, documented inline: `OSLogStore` only grants third-party apps access to their own current-process log entries. Quitting cairn before exporting wipes the in-process buffer. Recommended flow when triaging a bug: reproduce the issue → immediately go to Settings → Advanced → Export, without backgrounding cairn first.

Zero data collection on Anthropic's or maintainer's side — the file goes wherever the tester sends it, no servers in between. The export action also doesn't transmit anything itself; it just opens the iOS share sheet.

No changes to sync, indexing, or reconciliation behavior.
