Remove leftover resource-pick diagnostic instrumentation.

Builds 99-100 added a Logger.notice + NSLog pair fired from `resourcesToHash(for:)` and the imputation pass enumeration loop whenever a PHAsset's resource set carried both .photo+.fullSizePhoto or both .video+.fullSizeVideo — used to investigate the original "5 phantom .MOVs in Aging Out" hypothesis. That investigation concluded back at build 102 (resource selection isn't the cause; the actual fix was the alive-on-phone safety check in the reconciliation engine), but the diagnostic stayed in place running ~165 log lines per imputation pass.

This build strips it: removes the `OSLog` import, the `resourcePickLog` Logger, the `logResourceAmbiguity` helper, the `resourceTypeName` enum-pretty-printer, and both call sites (PhotoKitPhotoEnumerator.resourcesToHash and AppDependencies.runImputationPass). Pure observation code — no behavioral change, no test impact (662 tests pass unchanged).
