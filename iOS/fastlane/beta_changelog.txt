Fix: photos you viewed in Photos.app no longer pile up in the deferred queue. PhotoKit bumps the modificationDate for non-content events (iCloud sync after a view, album moves, favorite-toggle), and the previous filter treated any modDate divergence as "needs re-hash." On a device with iCloud Photos optimized storage, that led to a chain — view → modDate bump → re-hash → iCloud re-download → 60s timeout → defer with "timed out." The filter now does a secondary check: if a hash is already cached and the asset's primary resource size hasn't changed, treat the update as metadata-only churn and skip the re-hash. Real edits still re-hash because the editor adds a .fullSizePhoto adjustment resource that changes the size.
