Fix: the dark-mode color fix we shipped in build 77 for the "Never-touch ceiling" toggle (Settings → Library) didn't actually take. The replacement color we picked (`primaryInk`) resolves to the same near-white as the original under cairn's dark palette, so the pill background and the toggle's white thumb still blended together. Switched to cairn's verified-green (the same token the rest of the app uses for "this thing is on/active") so the on-state reads with proper contrast in both color schemes.

While I was in there, I made every other toggle in the app source its on-state from the same palette token rather than mixing the palette with iOS's default systemGreen. Toggles in Settings rows and in the "Find missed deletions" sheet now match the never-touch ceiling's on-state.

Also: toggling the Never-touch ceiling off and then back on now restores the value you had set instead of resetting to the 1 GB default. The previous value is remembered as long as you're in the same session.
