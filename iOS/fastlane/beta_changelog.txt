New: optional faster initial scan. Cairn can now look up phone photos in Immich's existing asset list (matching on the device-asset id that Immich's mobile uploader stamps at upload time) and trust the server-computed SHA1 instead of re-hashing locally. On iCloud-Optimized libraries this can drop the first scan from hours to seconds.

Default off — picking the path is a new onboarding step that explains both options neutrally. Existing installs see a new Settings → Initial scan section with a "Trust server checksums" toggle. The InitialScanScreen surfaces a separate "trust-seeded from server (skipped local hashing)" line during sync so it's always clear what cairn verified locally vs imputed.

Safety rails: only photos with an unambiguous, non-trashed server match get imputed (so Immich-app reinstall scenarios fall back to hashing for ambiguous rows). If <5% of phone photos match on a library of more than 100 assets, the optimization skips entirely — a fresh-phone-restore signal where local-identifier values rotated.

To test on an existing install without re-onboarding: Settings → Initial scan → toggle on, then Danger zone → Reset index → "This account," then trigger a sync. Watch the "Matching from server" phase, then the trust-seeded counter alongside the hashing one.
