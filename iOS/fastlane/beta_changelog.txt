Live progress during the phone-library scan portion of Matching from server.

The imputation pass has two slow steps: (1) enumerating every PHAsset to build the (filename, capture-date) join table, and (2) writing the matched entries into the local hash store. Build 101 added a counter for step 2, but for libraries large enough to make this matter, step 1 is what dominates wall-clock time (PHAssetResource.assetResources per asset is a few ms; 17k assets is ~30-60 seconds with zero UI signal).

This build streams a "scanned / total" count from inside the PHAsset enumeration loop (every 200 assets, fire-and-forget MainActor hop) into the existing syncProgress.hashed/total fields. The InitialScanScreen's "X / Y processed" line now ticks live throughout the Matching from server phase instead of sitting at 0 / 0 until the entire scan completes.

No behavior change to imputation or reconciliation.
