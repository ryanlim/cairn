Build-110 upgrade fix: existing installs auto-disable Keep-screen-on after upgrade.

Build 110 added a default-on `keepScreenAwakeDuringSync` toggle calibrated for the first sync, with an auto-disable that fires on the `hasCompletedInitialScan: false → true` transition. Users upgrading from a pre-110 build whose initial scan was *already complete* never hit that transition, so the toggle stayed on indefinitely and burned battery on every incremental sync.

This build adds a one-shot UserDefaults-gated migration that runs once during bootstrap. If the upgrade is detected (initial scan already complete + toggle still at factory default `true` + migration flag not yet set), cairn flips the toggle off, persists, and sets the flag. The fresh-install case is unaffected — on a new install the auto-disable still happens at first-scan completion as designed.

The migration flag also gets set unconditionally on first bootstrap of this build, so a user who manually re-enables Keep-screen-on after the upgrade isn't blindsided by the migration re-firing on a subsequent bootstrap. Manual re-enables persist as expected.

If your initial scan was already complete when you upgraded to 110 and you've been seeing the screen stay on through incremental syncs since, 111 will silently flip it off the next time the app launches. Re-enable from Settings → Library → Initial scan for any big follow-up run.
