"Open in Immich" now opens the Immich app directly when it's installed.

Previously this always opened the web viewer in a browser. Immich's iOS app registers a deep-link scheme (immich://asset?id=…), so cairn now tries that first and falls back to the browser (the server's /photos/<id> page) when the app isn't installed.

To test: open a run in the Runs tab, select an asset, tap "Open in Immich."
- With the Immich app installed → it should jump straight into the Immich app's asset viewer.
- Without it → same browser behavior as before.

Note: the deep link opens the asset in whichever server the Immich app is signed into (the link itself carries no server address — that's how Immich's own deep links work). Fine for a single-server setup; the browser fallback stays pointed at your specific server.

No other changes in this build.
