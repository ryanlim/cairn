Auto-recovery on session expiry. If Immich revokes your session token (admin action, server-side data loss, or a manual logout from the Immich web UI's Sessions list) and cairn's next /sync/* call returns 401, the app now: drops the dead token from Keychain, falls back to paginated discovery for the in-flight sync (so it doesn't fail outright), and surfaces a "Session expired" banner on the Status screen with a Sign-in-again button that opens the same session sheet you used for the first login. Sign back in to restore incremental sync.

Previously the dead token stayed in Keychain and every subsequent sync silently fell back to paginated with no indication anything had changed — a soft failure that was hard to notice. The banner makes it visible at a glance.

The detection only fires when a 401 lands on /sync/* and a session token was in use. A 401 from anywhere else (a revoked API key, for example) keeps its existing "auth expired" banner path — unchanged.
