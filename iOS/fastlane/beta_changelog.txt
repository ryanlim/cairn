App Store 0.4.0 release candidate (supersedes build 132).

Same as 132 plus one fix: the "Indexed" stat no longer dips during a sync and then settles to a different number. It flipped between two internal counts — a per-account/scope-filtered count and the global cache count the sync settles on. It now uses the global count consistently, so "Indexed" only grows as hashing proceeds, no transient drop.

Everything since the last App Store release (v0.3.1) is in the App Store "What's New":
- Faster first scan (trust-from-server).
- Stronger deletion safety (never trash a server photo still on your phone).
- A broad correctness + performance review.
- Clearer sync progress.
- Reworked Settings.
- Smaller: old-delete cutoff, Open-in-Immich deep link, opt-in diagnostic logging, hash cache in backup/restore.
