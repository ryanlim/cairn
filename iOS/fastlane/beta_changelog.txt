Sync journal: each row now shows what triggered the sync — trigger=manual, trigger=background, trigger=shortcut, trigger=debug — so you can verify background slots are firing as iOS schedules them. Manual and Shortcut taps always log, even when nothing changed, so you can confirm a button press registered. Three or more consecutive background syncs fold into a single tap-to-expand bundle with a time range, keeping the journal readable when many short slots run in succession.

Keychain bootstrap: more robust against transient Keychain errors at app launch. Earlier builds occasionally bounced you back to onboarding after a flaky launch even though server credentials were saved. cairn now distinguishes "credentials genuinely missing" from "Keychain hiccup, try again," retries up to five times before deciding the session is gone, and preserves the existing session instead of silently re-flowing onboarding.

Local Network reliability: declared the Local Network usage description that iOS now requires for connections to home-LAN servers, and added an App Transport Security exception so cairn can reach Immich servers exposed over plain HTTP on private IP ranges (the typical self-hosted setup). Without these, iOS could silently block the connection before cairn ever saw it as a network error.

Background sync label: the long-budget iOS background slot used to show "trigger=overnight" in the journal. iOS actually fires those slots whenever the device is charging, idle, and on Wi-Fi — daytime included — so the label was asserting a time-of-day it couldn't verify. All background syncs now consistently show "trigger=background".
