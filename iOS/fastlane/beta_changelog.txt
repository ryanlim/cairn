Two related additions for power-users with large libraries.

Settings → Data → Export data now includes the SHA1 hash cache (the localId→checksum map that took the longest to build on first scan). Import restores it automatically — but only when the device fingerprint matches. Backups taken on a different iPhone, or on this iPhone before a restore-from-backup, will silently skip the hash cache portion (observed history, exclusions, journal, and settings still come through). The backup's device fingerprint is the per-vendor identifierForVendor; it rotates on the same triggers that invalidate PhotoKit localIds, so a match is a hard guarantee that every cached row still resolves.

Settings → Safety rails → "Clear hash cache" is a new targeted reset: drops the SHA1 cache + change-tracking baseline + deferred queue, but leaves observed history, active quarantine, exclusions, and credentials intact. Lighter than Reset index. Useful for exercising the fast initial scan on an already-indexed library: export your cache first, clear it, watch the "Matching from server" phase and trust-seeded counter run, then re-import to restore.
