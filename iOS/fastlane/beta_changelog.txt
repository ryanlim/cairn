Settings → Initial scan now shows a cache breakdown when at least one row was trust-seeded from the server ("N verified · M trust-seeded"), and exposes a "Re-hash imputed entries" action that drops the trust-seeded rows so the next sync re-hashes them locally. Verified rows are preserved by the existing modDate-skip path, so the cost is bounded to just the imputed subset — though on iCloud-Optimized libraries each dropped row still needs its original downloaded.

The verify action is meant for users who want every checksum computed by cairn itself, not trusted from the server. Most people won't need it; the trust-seeded path is safe (server-computed SHA1s match cairn's algorithm, and any deletion that resolves through an imputed row still goes through Immich's 30-day Trash window).
