Import no longer silently overwrites your in-app settings.

Settings → Data & recovery → Import data was hard-wired to apply the settings block from the imported payload, overwriting whatever toggles you had set in the app. The behavior was visible during fast-initial-scan testing: clearing the hash cache and re-importing an older backup silently reverted the "Trust Immich's hashes" toggle to its pre-feature default, because the backup was created before the toggle existed.

Now: import restores the cache, observed checksums, exclusions, journal, and per-server state from the payload (the parts you actually want when restoring), but preserves your current in-app settings (toggles, thresholds, indexing scope, etc). The action remains otherwise identical.

Settings restoration during cross-device migration was the original intent of bundling settings into the export, but that flow doesn't go through this UI affordance anyway — the migration path runs through onboarding's "Restore from another device" route.
