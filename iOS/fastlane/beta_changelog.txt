Reliability: persistent fix for the transient onboarding-screen flash on launch. The retry-window extension we shipped in build 73 turned out to be masking a deeper issue: cairn was storing credentials with iOS's strictest Keychain accessibility tier (`WhenUnlocked`), which returns "item not found" any time the device is locked, including the brief locked transitions around background sync wake-ups and quick post-unlock launches. cairn now uses `AfterFirstUnlockThisDeviceOnly` instead, so credentials become readable after the first device unlock since boot and stay accessible through screen-lock cycles. A one-time migration on next launch upgrades the accessibility tier for credentials written by earlier builds; no user action required. This is the same fix Apple Developer Forums and apps like Square's Valet have documented for the same symptom.

UI fix: the "Never-touch ceiling" toggle (Settings → Library) rendered as a solid white pill with no visible thumb in Dark Mode. The toggle tint was the foreground text color, which collapsed the on-state into an indistinguishable filled bar. It now uses the same primary accent the rest of the Settings toggles do, with proper contrast in both color schemes.
