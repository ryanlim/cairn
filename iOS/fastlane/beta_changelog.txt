Keep-screen-on toggle for long-running syncs, on by default for the first scan.

Initial syncs on large libraries (50k+ assets) can run for tens of minutes — long enough that Auto-Lock kicks in, iOS suspends cairn after its ~30-second background grace, and the foreground hashing pass stalls until the user wakes the device. The tester then sees what looks like a stuck sync, and progress appears to "restart" when they relaunch (it's actually resuming from cache, but the UI doesn't telegraph that yet).

This build adds `keepScreenAwakeDuringSync` to settings. When on, cairn holds the iOS idle timer for the duration of a foreground sync via `UIApplication.shared.isIdleTimerDisabled = true`. Auto-Lock is suppressed; manual lock (side-button press) and app-switching still work. Cleared on every sync exit path (success, cancel, error) and force-cleared on scene-phase → background.

Default behavior:
- Factory default ON, so a fresh install's first sync runs without lock-pause interruptions.
- After the first sync's `hasCompletedInitialScan` flips true, cairn auto-flips this toggle off — incremental syncs are seconds-long and don't justify the battery cost.
- Users can re-enable from Settings → Library → Initial scan → "Keep screen on during sync" for any later big run (post-onboarding restore, bulk delete, etc). Manual re-enables persist; the auto-disable only happens once on first-ever completion.

Surfaced in two places: Initial scan screen → Scan options card (so the user sees it during onboarding), and Settings → Library → Initial scan (post-onboarding).
