Fix: the fast-initial-scan / imputation pass was running on every sync, not just on bootstrap. On normal incremental syncs the local cache is already populated so the imputation has nothing to seed — it'd find every phone asset as "already cached" and exit. But it still spent time enumerating the phone library + PHAssetResource filenames + the server (filename, capture-date) map, and worse, serialized the server fetch before the hash pass for that whole window. The "Matching from server" phase appearing briefly on every sync was the visible symptom.

Now gated on the PhotoKit change-token: imputation runs only when the token is absent (fresh install, after Clear Hash Cache, after Reset Index, after Rescan Library — the cases where the cache is actually empty). On normal syncs the server fetch runs parallel with the hash pass again as it should, and the "Matching from server" phase doesn't appear at all.
