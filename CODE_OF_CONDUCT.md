# Code of Conduct

cairn is a small project. The contributor norms here are short and
plain:

- **Treat people with respect.** Disagree with ideas, not people.
  Critique on the substance, not on character, identity, or
  experience level.
- **Assume good faith.** Most miscommunications are
  miscommunications, not malice. If a comment lands wrong, ask
  before escalating.
- **Be inclusive.** Don't gatekeep on background, role, fluency, or
  tooling. People showing up to help should get help.
- **Stay on topic.** Issues and PRs are for the work. Off-topic
  side discussions belong in their own threads or off the project.
- **No harassment.** Personal attacks, slurs, sustained disruption,
  doxxing, sexual attention, and similar behavior are out of bounds.
  This applies in issues, PRs, code comments, commit messages, and
  any other space tied to this project.

## Reporting

If something here happens to you or you see it happen to someone
else, open a [GitHub issue](https://github.com/glarue/cairn/issues)
or, for sensitive matters, contact the maintainer (`graham@glarue.net`).

The maintainer reviews reports privately, responds within a week,
and acts on them — that can mean editing or removing content,
locking threads, or removing contributors from the project. No
guaranteed process: this project is one person plus volunteers,
not a foundation.

## Scope

This applies to all project spaces: GitHub issues / PRs / discussions,
commit messages, code comments, and any communication that identifies
itself with cairn. Off-project conduct that materially affects the
project's ability to function (e.g. ongoing harassment of contributors)
is also in scope.
